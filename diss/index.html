Введение

Моделирование поведения интеллектуальных агентов в трёхмерном пространстве является одним из ключевых направлений современной прикладной математики и информатики. Актуальность этой темы обусловлена широким применением агентных моделей в различных областях — от робототехники и виртуальной реальности до компьютерных игр, транспортных систем и управления беспилотными аппаратами [1].

Имитация поведения агентов позволяет исследовать сложные системы, состоящие из множества взаимодействующих элементов, и прогнозировать их эволюцию в разнообразных условиях окружающей среды.

Целью данной работы является разработка математической модели симуляции поведения агентов в трёхмерной среде и проведение эксперимента по базовому сценарию эвакуации из здания. В процессе работы были изучены основные определения в сфере агентного моделирования, алгоритмы навигации и моделирования группового поведения, а также средства и процесс разработки модели поведения агентов.

В практической части была разработана модель, позволяющая пользователям проводить эксперименты по базовому сценарию эвакуации агентов из здания. Программа реализована на языке C# в среде Unity.

1. Анализ основных понятий в сфере агентного моделирования

1.1 Агент

Агент (интеллектуальный агент) — это сущность, воспринимающая окружающую среду посредством датчиков и воздействующая на неё через исполнительные механизмы [2]. Рациональный агент действует таким образом, чтобы максимизировать ожидаемое значение показателя эффективности на основе прошлого опыта и знаний [1].

Рисунок 1 — Агенты взаимодействуют с окружающей средой через датчики и приводы

Математически агент может быть описан функцией:

f:P* → A (1)

где P* — последовательность восприятий; A — множество возможных действий [2].

1.2 Классификация агентов

Агенты классифицируются на пять основных типов:

  1. Простые рефлекторные агенты — действуют на основе текущего восприятия
  2. Рефлекторные агенты на основе моделей — учитывают историю восприятий и внутреннее состояние
  3. Целевые агенты — стремятся к достижению заданной цели
  4. Агенты на основе полезности — максимизируют функцию полезности
  5. Обучающиеся агенты — адаптируются к среде через механизмы обучения [1]

1.3 Многоагентные системы

Многоагентные системы (MAS) состоят из группы слабо связанных автономных агентов, взаимодействующих для достижения общей цели [3]. Классификация MAS включает:

  • Однородные и гетерогенные системы [4]
  • Иерархические, холонические, коалиционные и командные архитектуры [6, 7, 8]

2. Методы и инструменты для моделирования

2.1 Выбор программного обеспечения для моделирования

Для реализации модели были рассмотрены следующие платформы:

  • Unity3D — мощный движок для 3D-визуализации и физики, поддержка C# [10]
  • Unreal Engine — фотореалистичная графика, визуальное программирование через Blueprints [11]
  • Blender — бесплатное ПО с поддержкой Python, но ограниченной документацией
  • AnyLogic — специализированное ПО для агентного моделирования, но с ограниченной 3D-визуализацией [12]

Выбор пал на Unity3D благодаря гибкости, поддержке физики NVIDIA PhysX и возможности интеграции с ML-Agents.

2.2 Математическая модель симуляции поведения

Для моделирования поведения агентов в 3D-пространстве разработана формальная модель, включающая несколько взаимосвязанных компонентов. Рассмотрим ключевые компоненты.

Первая компонента — формализация агента. Каждый агент представляется как объект с набором параметров:

  • позиция в пространстве: ri = (xi, yi, zi), задающая текущие координаты агента
  • скорость: vi(t) = dri/dt и направление движения (обычно единичный вектор ni), определяющие динамику перемещения
  • целевая точка или маршрут — например, список координат, через которые агент планирует пройти G = {g1, g2, ..., gn}, либо более сложное представление цели (выйти из здания, достичь определённой области)
  • внутренние состояния (при необходимости): уровень энергии или «здоровья», статус (например, «эвакуируется», «паника»), роли (обычный агент или лидер группы) и т.д.

Вторая компонента — механика движения. Используется модель социальных сил Хелбинга [13]:

midvi/dt = Fi(goal) + Σj≠iFij(rep) + ΣkFik(wall) (2)

где:

  • midvi/dt — инерция (масса mi и ускорение агента)
  • Fi(goal) — сила стремления к собственной цели (направленная к ближайшей точке маршрута или выходу и стремящаяся разогнать агента до желаемой скорости)
  • Fij(rep) — силы отталкивания от других агентов j (предотвращают столкновения, моделируют личное пространство)
  • Fik(wall) — силы отталкивания от препятствий k (стены, объекты в окружении)

Третья компонента — алгоритмы навигации:

  • A* — для поиска кратчайшего пути [14]
  • RRT — для планирования в динамической среде [15]
  • Метод потенциальных полей — для управления движением

Четвертая компонента — моделирование группового поведения:

  • Правила роя (Boids) — сближение, выравнивание, разделение [16]
  • Теория игр — для моделирования кооперации и конкуренции

Пятая компонента — интеграция искусственного интеллекта:

  • Конечные автоматы (FSM) — для управления состояниями агентов
  • Деревья принятия решений — для иерархического выбора действий
  • Обучение с подкреплением (RL) — для адаптивного поведения [17]

3. Практическое моделирование

3.1 Описание экспериментальной среды

Экспериментальная среда моделирует эвакуацию из здания в Unity3D. Параметры агентов включают:

  • Скорость движения: 1-3 м/с
  • Радиус личного пространства: 0.3-0.4 м
  • Уровень паники и склонность к следованию за лидером

Рисунок 7 — Визуализация объема зрения агента

Сценарии экспериментов:

  1. базовый сценарий: эвакуация без внешних угроз
  2. динамические препятствия: обрушение части здания
  3. введение паники: распространение ложной информации через агентов
  4. наличие координаторов (управляемая эвакуация)

Метрики анализа:

  • время эвакуации в зависимости от количества агентов
  • процент успешно вышедших из здания
  • плотность толпы в ключевых точках (дверные проёмы)

Визуализация и сбор данных:

  • режим наблюдения: свободная камера, тепловые карты плотности
  • экспорт данных: логи позиций агентов в CSV-формат для анализа в Python (Matplotlib, Pandas)

3.2 Проведение начальной симуляции

В базовом сценарии 100 агентов эвакуировались из здания через два выхода за ~30 секунд. Заторы наблюдались только у дверных проёмов, но не приводили к полной остановке потока.

Заключение

В ходе работы была разработана математическая модель симуляции поведения агентов в трёхмерной среде и реализована программа для проведения экспериментов по эвакуации. Модель учитывает физические ограничения, взаимодействие агентов и элементы искусственного интеллекта.

Полученные результаты могут быть использованы для решения задач планирования эвакуации, оптимизации движения транспорта и моделирования толп в общественных пространствах. Дальнейшие исследования могут быть направлены на интеграцию методов глубокого обучения с подкреплением для повышения автономности агентов.
