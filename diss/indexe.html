Introduction

Modeling the behavior of intelligent agents in three-dimensional space is one of the key areas of modern applied mathematics and computer science. The relevance of this topic is due to the wide application of agent-based models in various fields - from robotics and virtual reality to computer games, transportation systems, and unmanned aerial vehicle control [1].

Simulating agent behavior allows researchers to study complex systems consisting of multiple interacting elements and predict their evolution in diverse environmental conditions.

The purpose of this work is to develop a mathematical model for simulating agent behavior in a three-dimensional environment and conduct an experiment on a basic building evacuation scenario. During the work, the main definitions in the field of agent-based modeling, navigation algorithms, and group behavior modeling were studied, as well as the tools and process for developing agent behavior models.

In the practical part, a model was developed that allows users to conduct experiments on the basic scenario of agent evacuation from a building. The program was implemented in C# in the Unity environment.

1. Analysis of Basic Concepts in Agent-Based Modeling

1.1 Agent

An agent (intelligent agent) is an entity that perceives the environment through sensors and acts upon it through actuators [2]. A rational agent acts in such a way as to maximize the expected value of a performance measure based on past experience and knowledge [1].

Figure 1 — Agents interact with the environment through sensors and actuators

Mathematically, an agent can be described by a function:

f:P* → A (1)

where P* is a sequence of perceptions; A is a set of possible actions [2].

1.2 Agent Classification

Agents are classified into five main types:

  1. Simple reflex agents — act based on current perception
  2. Model-based reflex agents — consider perception history and internal state
  3. Goal-based agents — strive to achieve a given goal
  4. Utility-based agents — maximize a utility function
  5. Learning agents — adapt to the environment through learning mechanisms [1]

1.3 Multi-Agent Systems

Multi-agent systems (MAS) consist of a group of loosely coupled autonomous agents that interact to achieve a common goal [3]. MAS classification includes:

  • Homogeneous and heterogeneous systems [4]
  • Hierarchical, holonic, coalitional, and team architectures [6, 7, 8]

2. Methods and Tools for Modeling

2.1 Selection of Simulation Software

The following platforms were considered for model implementation:

  • Unity3D — powerful engine for 3D visualization and physics, C# support [10]
  • Unreal Engine — photorealistic graphics, visual programming through Blueprints [11]
  • Blender — free software with Python support, but limited documentation
  • AnyLogic — specialized software for agent-based modeling, but with limited 3D visualization [12]

Unity3D was chosen due to its flexibility, NVIDIA PhysX physics support, and ML-Agents integration capability.

2.2 Mathematical Model of Behavior Simulation

For simulating agent behavior in 3D space, a formal model was developed, including several interconnected components. Let's consider the key components.

The first component — agent formalization. Each agent is represented as an object with a set of parameters:

  • position in space: ri = (xi, yi, zi), defining the agent's current coordinates
  • velocity: vi(t) = dri/dt and direction of movement (usually a unit vector ni), determining movement dynamics
  • target point or route — for example, a list of coordinates that the agent plans to pass through G = {g1, g2, ..., gn}, or a more complex goal representation (exit the building, reach a certain area)
  • internal states (if necessary): energy or "health" level, status (e.g., "evacuating", "panic"), roles (regular agent or group leader), etc.

The second component — movement mechanics. Helbing's social force model is used [13]:

midvi/dt = Fi(goal) + Σj≠iFij(rep) + ΣkFik(wall) (2)

where:

  • midvi/dt — inertia (mass mi and agent acceleration)
  • Fi(goal) — force striving toward the agent's own goal (directed toward the nearest route point or exit and aiming to accelerate the agent to the desired speed)
  • Fij(rep) — repulsive forces from other agents j (prevent collisions, model personal space)
  • Fik(wall) — repulsive forces from obstacles k (walls, objects in the environment)

The third component — navigation algorithms:

  • A* — for shortest path finding [14]
  • RRT — for planning in dynamic environments [15]
  • Potential fields method — for movement control

The fourth component — group behavior modeling:

  • Swarm rules (Boids) — cohesion, alignment, separation [16]
  • Game theory — for modeling cooperation and competition

The fifth component — artificial intelligence integration:

  • Finite state machines (FSM) — for agent state management
  • Decision trees — for hierarchical action selection
  • Reinforcement learning (RL) — for adaptive behavior [17]

3. Practical Modeling

3.1 Description of the Experimental Environment

The experimental environment simulates building evacuation in Unity3D. Agent parameters include:

  • Movement speed: 1-3 m/s
  • Personal space radius: 0.3-0.4 m
  • Panic level and tendency to follow a leader

Figure 7 — Visualization of agent field of view

Experiment scenarios:

  1. basic scenario: evacuation without external threats
  2. dynamic obstacles: partial building collapse
  3. introducing panic: spreading false information through agents
  4. presence of coordinators (managed evacuation)

Analysis metrics:

  • evacuation time depending on the number of agents
  • percentage of agents successfully exiting the building
  • crowd density at key points (doorways)

Visualization and data collection:

  • observation mode: free camera, density heat maps
  • data export: agent position logs in CSV format for analysis in Python (Matplotlib, Pandas)

3.2 Conducting the Initial Simulation

In the basic scenario, 100 agents evacuated from the building through two exits in ~30 seconds. Congestion was observed only at doorways, but did not lead to a complete stop of the flow.

Conclusion

During the work, a mathematical model for simulating agent behavior in a three-dimensional environment was developed and a program for conducting evacuation experiments was implemented. The model takes into account physical constraints, agent interactions, and artificial intelligence elements.

The obtained results can be used to solve problems of evacuation planning, traffic flow optimization, and crowd modeling in public spaces. Further research may be directed toward integrating deep reinforcement learning methods to increase agent autonomy.
